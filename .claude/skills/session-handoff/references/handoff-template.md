# Session Handoff Template

Use this template for `docs/session/SESSION_HANDOFF.md`.
Every section is required. Write "None" if a section has no items.

---

## Last Updated

`YYYY-MM-DD` -- Set to the current date each time the handoff is updated.

## Session Summary

Two to four sentences describing the focus and outcome of this session.

## Work Completed

- Bulleted list of items finished this session.
- Include file paths or component names where relevant.

## Work In Progress

- Bulleted list of started-but-unfinished items.
- Note current state and what remains for each.

## Blocked Items

- Items that cannot proceed without external input.
- State what the blocker is and who can unblock it.

## Next Steps

1. Prioritized list of what to tackle next.
2. Include enough context to start immediately.
3. Reference relevant files, plans, or issues.

## Key Decisions Made

- Decisions made this session that affect future work.
- Include brief rationale; link to ADR if applicable.

## Open Questions

- Unresolved questions that need answers.
- Note who might have the answer, if known.

## Files Modified

- List of files created or modified this session.
- Use relative paths from the project root.
